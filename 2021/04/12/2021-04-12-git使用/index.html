    Git使用

    发表于 04-12-2021 更新于 04-13-2021 阅读次数： Disqus：
    git

    记录

    工作区 是创造文件的地方 随意修改本地库都不会改变

    版本库 是git管理所在 工作区修改add commit后 本地库就修改了

    远程库 git远程代管 push到远程库 远程库修改

    Git

    创建仓库git init

    • git init 初始化一个文件夹为仓库

    • git init <repositoryName> 创建并且初始化一个自定义名称的文件夹为仓库

    创建目录（文件夹） 或 文件

    • mkdir <directoryName> 创建文件夹

    • touch <filmName> 创建文件

    • 编辑文件

      • 手动打开编辑器以编辑文件
      • start <fileName> 命令行打开默认编辑器以编辑文件

    • 撤销编辑

      • 手动删除
      • git checkout -- <fileName> 命令行删除

    • 删除文件

      • 手动删除
      • rm <fileName> 命令行删除

    增加文件进入暂存区git add

    • git add <.> 添加当前目录下所有文件夹和文件
    • git add <directoryName> 添加一个文件夹的内容
    • git add <fileName> 添加一个文件内容
    撤销git add
    • git reset <fileName>

    提交文件进入版本库git commit

    • git commit -m “message” 提交文件并且写下提交注释信息
    • 每个commit 都有一个 commit_id
    回退 commit

    • git reset --hard <options>

      • options包括

        1.通过HEAD指针指向某一个版本

        • 例如：HEAD^ “上一个版本” HEAD^^ “上上一个版本” HEAD~100 “上一百个版本”

        2.通过commit_id 定位某个版本

    从版本库删除文件
    • git rm <fileName> 先删除
    • git commit -m "message" 后提交删除记录
    从版本库中恢复文件
    • git checkout -- <fileName>

    将本地电脑连接远程服务器github

    • 需要获取ssh密钥将自己的电脑连接上github
      • ssh-keygen -t rsa -C email

    • 去本地c盘目录下获取密钥 复制粘贴至github
    • 检查连接是否成功
      • ssh -T git@github.com
    • git remote add <remoteRepositoryName> <url> 增加一个远程库

    将提交至版本库的文件部署至github

    • git push <name> <local branch> : <remote branch>将本地分支提交github一个分支

    分支

    • git branch <newBranchName> 创造分支
    • git checkout <branchName> 切换分支
      • 或者使用 git switch <branchName>
    • git checkout -b <newBranchName> 创造并切换分支
      • 或者使用 git switch -c <newBranchName>
    • git branch 查看分支目录
    • git merge <branchName> 在一分支里合并所指定的分支
    • git branch -d <branchName> 删除分支
